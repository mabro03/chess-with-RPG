# 프로젝트 제목: chess-with-RPG

## 구현된 핵심 시스템
### 기본적인 체스 매커니즘
+ 규칙 구현
  + 표준 이동 규칙: 킹, 퀸, 룩, 비숍, 나이트, 폰의 표준적인 이동 및 공격 규칙을 모두 구현하였습니다.

  + 체크 및 체크 메이트: 기본적인 체크 및 체크메에트 판정 로직을 따로 구현하지 않고, 킹의 HP가 0이 될 경우 승리하는 새로운 규칙으로 대체하여 기존 체스게임의 구조를 재설정하였습니다.

- 게임 구조: 턴제 시스템입니다. 플레이어는 백색, ai는 흑색으로 정하였고, 이 둘이 번갈아 가며 턴을 진행하는 기본적인 턴제 구조를 구현하였습니다.

 * 8*8 표준 체스 보드를 Ptgame의 Grid 시스템을 이용하여 구현하였고, 각 기물의 위치를 좌표계로 관리합니다. 

### 스탯 및 전투 시스템
+ 스탯 정의: 모든 기물에 체력(HP)와 공격력(AP) 스탯을 부여했습니다.
  + 기물의 스탯을 정리하면 아래와 같습니다.
  + 킹 -> 체력:20, 공격력:5 
  + 퀸 -> 체력:15, 공격력:9
  + 룩 -> 체력:12, 공격력:7
  + 비숍 -> 체력:10, 공격력:6 
  + 나이트-> 체력:9, 공격력:6 
  + 폰 -> 체력:5, 공격력:3
 
+ 전투 메커니즘: 기물이 상대방의 기물이 위치한 곳에 이동하려할 때 전투가 일어나게끔 했습니다.
  + 데미지 계산: 데미지의 계산의 복잡도를 낮추고자 방어력이 아닌 일시적인 추가체력으로 방어도를 고려하였습니다.
  + 기물의 제거 조건: 일반적인 체스와 다르게 체력(HP)이 부여된 만큼 이 수치가 0이 되어야 보드에서 제거됩니다.

+ 승리 조건: 기존 체스 게임의 체크메이트가 아닌 상대방의 킹의 HP를 0으로 만들어야 됩니다. 

# 체스의 기물별 특수 능력과 매커니즘
+ 각 기물에 부여된 특수 능력은 아래와 같습니다.
  + 킹: 매턴 시작 시 체력을 4만큼 회복합니다. 
  + 퀸: 공격 대상 기물을 타격한 뒤 해당 기물의 뒤쪽에 위치한 기물까지 타격합니다.(쿨타임 3턴) 
  + 룩: 이동을 마칠 경우 다음 턴까지 임시의 체력 3만큼을 스스로에게 부여합니다. 
  + 비숍: 이동을 마친 칸을 기준으로 주변 3*3 범위의 아군 기물에게 체력 3을 회복시킵니다. 
  + 나이트: 앞에 기물이 있어도 넘어서 공격이 가능합니다.(기존의 특성이 너무 강력하여 능력이 없습니다.)
  + 폰: 게임 시작후 첫 공격을 할 때 1의 추가 공격력을 얻습니다.

# AI 구현 방식
 + 간소화된 미니맥스 알고리즘 기반이 턴제 AI를 구현하였습니다.
   + 이 AI는 단순 기물 교환 가치뿐만 아닌, 현재 기물의 체력과 공격력을 고려하여 가장 유리한 이동을 선택하도록 수정하였습니다.  

## 실행 방법 및 데모
### 환경 설정
+ 환경 설정
  + Python 필요 버전: 프로젝트는 Python 3.x 환경에서 개발되었습니다.
  + 꼭 필요한 라이브러리: 본 프로젝트는 Pygame 라이브러리를 사용합니다. 아래 명령어를 통해 라이브러리를 설치해야합니다.
    + Bash
    + pip install pygame

### 실행 방법
프로젝트 파일을 실행하는 명령어와 유의사항입니다. 
+ 파일 준비: 업로드되어있는 ZIP 파일을 해제한 후, 소스 파일이 포함된 최상위 디렉토리로 이동합니다.
+ 실행 명령어: 터미널에서 다음 명령어를 입력하여 게임을 실행합니다.
  + Bash
  + python chess_source_code.py
